# FoodOrderApp — SwiftUI Food Ordering Application

## Описание

Приложение для заказа еды в Минске. Агрегатор ресторанов с поддержкой регистрации пользователей, выбором блюд, корзиной, картой и несколькими способами оплаты.

## Шаги создания проекта в Xcode

### 1. Создание нового проекта

1. Открыть Xcode → **File → New → Project**
2. Выбрать **iOS → App**
3. Заполнить:
   - **Product Name:** `FoodOrderApp`
   - **Team:** Ваш Apple ID / команда
   - **Organization Identifier:** `com.yourname`
   - **Bundle Identifier:** `com.yourname.FoodOrderApp`
   - **Interface:** SwiftUI
   - **Language:** Swift
   - **Storage:** None (мы используем SQLite вручную)
4. Нажать **Next**, выбрать папку, нажать **Create**

### 2. Настройка структуры папок в Xcode

В Navigator (левая панель) создать группы (New Group):
- `Models`
- `Database`
- `ViewModels`
- `Views`

### 3. Добавление файлов

Либо:
- **Вариант А:** Скопировать содержимое папки `FoodOrderApp/` из этого архива поверх сгенерированной папки проекта, затем перетащить файлы в Xcode и выбрать "Create groups"
- **Вариант Б:** Создать каждый файл вручную через **File → New → File → Swift File** и вставить код

### 4. Добавление SQLite framework

В настройках таргета:
1. Выбрать таргет `FoodOrderApp`
2. Вкладка **General → Frameworks, Libraries, and Embedded Content**
3. Нажать **+**
4. Найти `libsqlite3.tbd` и добавить

> ⚠️ Это критически важно! Без этого шага приложение не скомпилируется.

### 5. Разрешения геолокации (Info.plist)

Добавить ключ в Info.plist (или через Build Settings → Info):
```xml
<key>NSLocationWhenInUseUsageDescription</key>
<string>Для определения ближайшего ресторана</string>
```

Если используется `GENERATE_INFOPLIST_FILE = YES` (как в нашем проекте), ключ уже прописан в Build Settings через `INFOPLIST_KEY_NSLocationWhenInUseUsageDescription`.

### 6. Если используется этот архив напрямую

```bash
# Распаковать архив
unzip FoodOrderApp.zip -d ./

# Открыть проект
open FoodOrderApp/FoodOrderApp.xcodeproj
```

После открытия в Xcode:
1. Выбрать свой Development Team в **Signing & Capabilities**
2. Убедиться, что `libsqlite3.tbd` добавлен (шаг 4)
3. Выбрать симулятор iPhone (например iPhone 15)
4. Нажать **Run (▶)**

## Описание экранов

### 🔐 Экран авторизации (AuthView)
- SegmentedControl: **Вход / Регистрация**
- При **Входе**: поля Login + Password → проверка в БД SQLite
- При **Регистрации**: Login + Password + Адрес → запись в БД → вход
- Валидация: логин ≥ 3 символа, пароль ≥ 4 символа, адрес ≥ 5 символов
- Показ/скрытие пароля

### 🍽 Экран меню (MenuView)
- Категории: Пицца 🍕 | Бургеры 🍔 | Закуски 🍟 | Напитки 🥤
- Каждая категория: заголовок + сетка карточек блюд (2 колонки)
- Карточка: цветной градиент + эмодзи + название + цена
- Фиксированная кнопка **Корзина** внизу с суммой и количеством
- Кнопка выхода (logout) в шапке

### 🛍 Экран настройки блюда (DishDetailView)
- Кнопка **← Меню** (отменяет изменения)
- Героическое изображение блюда
- **Пицца**: размер (15/20/23 см), тесто (тонкое/обычное), вкус (острое/оригинальное) — SegmentedControl
- **Бургер**: чекбоксы — сыр, лук, огурчик, помидор
- **Закуска**: порция (большая/средняя/маленькая) — SegmentedControl
- **Напиток**: объём (0.3л / 0.5л) — SegmentedControl
- Фиксированная кнопка **Добавить в корзину**

### 🛒 Экран корзины (CartView)
- Список блюд с чекбоксами (включить/исключить из заказа)
- Кнопка удаления блюда (🗑)
- Динамически пересчитываемая сумма
- SegmentedControl способа оплаты: **Онлайн | ЕРИП | Терминал | Наличные**
- Адрес из профиля пользователя (из БД)
- Кнопка **Выбрать на карте**
- Поле для комментария к заказу
- Фиксированная кнопка **Заказать** → вывод заказа в консоль → сброс корзины → возврат в меню

### 🗺 Экран карты (MapView)
- Карта Минска с 5 маркерами ресторанов
- Нажатие на маркер → выбор ресторана (маркер становится оранжевым)
- Панель снизу: адрес выбранного ресторана
- Кнопка **Ближайший** — определяет геопозицию и выбирает ближайший ресторан
- Кнопка **Выбрать** — подтверждает выбор и возвращает адрес в корзину
- Кнопка **Отмена** — возврат без изменений

## Модели данных

### SQLite таблица `user`
| Поле   | Тип   | Ограничения        |
|--------|-------|--------------------|
| id     | TEXT  | PRIMARY KEY        |
| login  | TEXT  | NOT NULL, UNIQUE   |
| pass   | TEXT  | NOT NULL           |
| adress | TEXT  | NOT NULL           |

### Иерархия классов блюд
```
Dish (базовый класс)
├── Pizza    [size: Int, pastry: String, spicy: Bool]
├── Burger   [cheese, onion, cucumber, tomato: Bool]
├── Snack    [size: String]
└── Drink    [volume: Double]
```

### Order
```swift
class Order {
    var number: Int
    var items: [OrderItem]   // Dish + included flag
    var address: String
    var payment: String
    var comment: String
    var sumPrice: Double     // Computed, только included блюда
}
```

## Валидация полей

| Поле | Правило |
|------|---------|
| Login | Не пустой, ≥ 3 символа |
| Password | Не пустой, ≥ 4 символа |
| Address (signup) | Не пустой, ≥ 5 символов |
| Уникальность логина | Проверяется в SQLite перед созданием |
